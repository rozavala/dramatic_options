# Thesis-premise-currency check — a new §11 funnel stage (proof-of-need + proposed amendment)

**Date:** 2026-06-30 · **Status:** proposed `PREREG_UNIVERSE_CURATION §11` amendment (for operator
ratification) + the dated proof-of-need. The first two applications (AUR, ERII) both **dropped**.

## The gap AUR exposed (and ERII confirmed)

The §11 funnel is: theme register → **feasibility screen** (cap-fit ∩ liquidity ∩ achieved-OTM) →
register-draft → admit. **The feasibility screen is structurally blind to thesis-staleness** — it
tests *"can this name carry a cheap convex bet,"* not *"is the claim about this name still true."*
Those are orthogonal. Both 2026-06-29/30 candidates **passed feasibility cleanly** and were one pin
from the forward Brier record, yet **both carried a false premise:**

- **AUR (autonomous_freight)** — thesis: "market prices Aurora as a continuing single-lane pilot;
  the un-narrated repricing is commercial scale-out." **Current state:** already multi-lane (Dallas–
  Houston, El Paso–Phoenix, Fort Worth–Phoenix, Laredo–Dallas, Dallas–OKC), 250K+ driverless miles,
  200-truck target, and the stock pops on each milestone (+15.5% on the Dallas–Houston launch). →
  the "2nd-lane" falsifier leg is **pre-satisfied**; milestone price reactions = `under_narrated=False`.
  Caught **late, by luck** — only because anchoring the falsifier N forced a current-state pull.
- **ERII (desalination_efficiency)** — thesis: "narrative sits on the CO2-refrigeration pivot,
  leaving desal under-attended as RO capacity inflects." **Current state:** CO2-refrigeration
  **wound down** (Feb-25-2026 exit) → the displacement premise is *false*; 2026 desal guidance
  **down / delayed into 2027** → the inflection is *inverted*; Evercore + Raymond James cover it
  **Hold** → not attention-displaced. All three legs fail. Caught **deliberately** — the first
  application of the check, run *before* pinning.

**Net: zero admits this round.** Both feasibility-survivors failed the premise check.

## The proposed §11 amendment — a premise-currency stage

> **§11.x — Thesis-premise-currency check (gate, between feasibility-pass and register-draft).**
> Before a feasibility-passing candidate is drafted into the register, verify via a **current-state
> pull** (latest filings + recent news/coverage) that the **thesis premise is presently true**:
> (a) the named narrative/mechanism still holds (not exited/faded/inverted), and (b) the name is
> still **attention-displaced** (not actively covered + priced). A failed premise → **drop or
> re-thesis, never an automatic pass.** This is orthogonal to feasibility (which tests tradeability,
> not premise-currency) and to the council's `under_narrated` read (which judges only what reaches
> it); it is a pre-register gate so a stale-premise name is caught by the **funnel**, not by luck.
> Cheap (one pull per surviving candidate). Record the pull + disposition with the window.

## The durable lesson (weight this above the single admit it cost)

AUR didn't just fail — it failed **late**, past every existing gate, one pin from the forward record.
That's the signal the funnel was **missing a stage**, not that AUR was a one-off. The premise check
converts the near-miss into a permanent improvement. And the round's real finding: **generation is
the binding constraint** — the last two hunts (9 operator themes + the one-layer-down batch) produced
exactly one feasibility-survivor between them (ERII), and the premise check then dropped it too. The
scarce resource is the **supply of names that pass all three orthogonal filters: feasible AND quiet
AND premise-current.** That is where the next judgment cycle goes.
